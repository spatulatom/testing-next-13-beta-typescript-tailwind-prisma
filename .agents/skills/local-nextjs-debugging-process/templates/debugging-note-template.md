# Next.js Investigation Note

Use this note when a debugging pass spans multiple tools, phases, or sessions.

## Symptom

- What is failing or surprising?

## Phase Chosen

- Phase 1: `next dev`
- Phase 2: `next build`
- Phase 3: build output
- Phase 4: production

## Tool Used

- Command, MCP tool, browser tool, or file inspection

## Raw Evidence

- Exact log line, route classification, emitted marker, screenshot observation, or HTML snippet

## Interpretation

- What this evidence does prove
- What it does not prove

## Next Discriminating Check

- Smallest next step that can falsify the current hypothesis

## Root Cause Hypothesis

- One concrete explanation tied to source or framework behavior

## Fix

- Smallest change that should remove the cause

## Validation

- What command, route reload, build, or emitted-file check confirms the fix
